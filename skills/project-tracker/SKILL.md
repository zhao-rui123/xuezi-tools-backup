# 项目跟踪技能

## 跟踪方式

使用 Markdown 文件进行轻量级任务跟踪：

```
projects/
└── 项目名称/
    ├── README.md          # 项目概览
    ├── TODO.md            # 待办事项
    ├── CHANGELOG.md       # 变更记录
    └── notes/             # 开发笔记
```

## TODO.md 格式

```markdown
# 项目名称 - 待办事项

## 进行中
- [ ] 任务3 - 预计完成: 2026-03-05

## 待开始
- [ ] 任务4 - 优先级: 高
- [ ] 任务5 - 优先级: 中

## 已完成
- [x] 任务1 - 完成: 2026-03-03
- [x] 任务2 - 完成: 2026-03-04

## 阻塞项
- 任务6 - 等待: 外部API审批
```

## 任务优先级标记

| 标记 | 含义 | 处理顺序 |
|------|------|---------|
| 🔴 P0 | 紧急 | 立即处理 |
| 🟡 P1 | 高优先级 | 24小时内 |
| 🟢 P2 | 中优先级 | 本周内 |
| ⚪ P3 | 低优先级 | 有空时 |

## 与多Agent开发结合

### 任务分配模板
```markdown
## 任务: 添加电价数据导出功能

**需求描述**: 支持导出Excel格式的电价数据
**验收标准**:
- [ ] 支持31省份数据导出
- [ ] 包含分时电价和循环次数
- [ ] 文件格式正确，可正常打开

**分配**:
- Alpha: 前端导出按钮
- Bravo: 数据格式化逻辑
- Echo: 测试验证

**截止时间**: 2026-03-06
```

## 最佳实践

1. **每日更新**: 站会格式记录进度
2. **及时归档**: 完成的任务移到"已完成"
3. **保持简洁**: 一个项目的TODO不超过20项
4. **定期回顾**: 每周回顾项目状态看板
