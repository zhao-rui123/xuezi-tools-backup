## 核心优化策略

### 1. 数据缓存策略

#### 缓存结构
```json
{
  "stocks_basic": {
    "updated": "2026-03-13 15:30:00",
    "data": [
      {"code": "002460.SZ", "name": "赣锋锂业", "industry": "锂电池"},
      ...
    ]
  },
  "daily_prices": {
    "002460.SZ": {
      "updated": "2026-03-13",
      "data": [
        {"date": "20260313", "close": 69.5, "change": -0.95},
        ...
      ]
    }
  },
  "financial_indicators": {
    "002460.SZ": {
      "roe": 15.2,
      "pe": 25.3,
      "pb": 3.2
    }
  }
}
```

#### 更新频率
| 数据类型 | 更新频率 | 触发方式 |
|---------|---------|---------|
| 股票基础信息 | 每日1次 | 定时任务 (00:00) |
| 日线行情 | 每日1次 | 收盘后 (15:35) |
| 财务指标 | 每季度 | 财报发布后 |
| 实时行情 | 按需 | 用户查询时 |

### 2. Token 优化输入格式

**压缩率: ~70%**

### 3. 分批处理策略

#### 大批量筛选 (100+股票)
```python
# 分批处理，每批10只
batches = [stocks[i:i+10] for i in range(0, len(stocks), 10)]

for batch in batches:
    # 每批独立分析
    result = analyze_batch(batch)
    # 合并结果
    results.append(result)
```

**Token 节省: 避免单请求超出上下文限制**

### 4. 预计算指标

#### 服务端预计算
```python
# 每日收盘后预计算
precomputed_metrics = {
    "002460.SZ": {
        "trend": "up",           # 趋势
        "volatility": 2.5,       # 波动率
        "score": 75,             # 综合评分

    }

```

**优势**: 用户查询时直接返回预计算结果，无需实时调用大模型

## 具体实现方案

### 方案 A: 轻量级筛选 (推荐)

**适用场景**: 快速筛选、实时查询

**流程**:
1. 从缓存读取预计算指标
2. 按条件筛选 (ROE>15, PE<30等)
3. 返回结果 (0 Token)

**Token 消耗**: 0
**响应时间**: <100ms

### 方案 B: 智能分析

**适用场景**: 深度分析、投资建议

**流程**:
1. 筛选出候选股票 (10-20只)
2. 压缩数据格式
3. 调用大模型分析
4. 返回结构化报告

**Token 消耗**: 2000-4000
**响应时间**: 2-5s

### 方案 C: 批量报告

**适用场景**: 每日收盘报告

**流程**:
1. 盘后自动获取全市场数据
2. 预计算所有指标
3. 生成报告模板
4. 异步发送 (非实时)

**Token 消耗**: 10000+ (每日1次)
**响应时间**: 不敏感

## Token 消耗对比
