## 2026-03-04 飞书文件发送路径规范

**决策内容**:  
所有飞书文件发送必须使用 `~/.openclaw/workspace/` 目录，禁止使用 `/tmp/`

**决策原因**:
- OpenClaw 的 `message` 工具在 Mac 上发送 `/tmp/` 目录文件会失败
- 返回成功但实际未发送，难以排查
- 使用 workspace 目录可稳定工作

**影响范围**:
- 所有截图发送
- 所有文档发送
- 所有图片转发

**执行方案**:
1. 创建技能包 `feishu-image-send` 记录正确方法
2. 更新 MEMORY.md 添加双重保险
3. 截图命令: `/usr/sbin/screencapture -x ~/.openclaw/workspace/screenshot.png`

**验证结果**: ✅ 已验证截图和文档发送均正常
