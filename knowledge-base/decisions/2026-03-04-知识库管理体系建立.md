# 2026-03-04 知识库管理体系建立

**状态**: ✅已采纳
**标签**: #知识库 #管理 #决策

## 背景
之前所有知识都堆在 MEMORY.md 一个文件里，随着时间推移会越来越难维护。需要建立结构化的知识管理体系。

## 决策内容

### 建立四层知识库结构

```
knowledge-base/
├── projects/      # 项目知识库
├── decisions/     # 重要决策记录
├── problems/      # 问题与解决方案
├── references/    # 参考资料
└── operations/    # 运维知识（新增）
```

### 文档规范
- 所有文档必须带标签（#标签）
- 必须标注状态（✅已验证 / ⚠️试行 / ❓待验证）
- 使用统一的文档模板

### 维护规则
- **每日**：检查 INDEX.md 是否需要更新
- **每周**：回顾 decisions/，标记已验证的决策
- **每月**：归档旧文档

## 备选方案

| 方案 | 评估 | 结论 |
|------|------|------|
| 继续用 MEMORY.md | 不可持续 | ❌放弃 |
| 用 Notion/飞书文档 | 外部依赖 | ❌放弃 |
| 本地 Markdown 结构 | 可控、可版本管理 | ✅采纳 |

- 知识传承更系统
