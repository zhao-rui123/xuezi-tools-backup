# 参考资料 - 模型配置

## 当前模型配置

| 模型 | Provider | 特点 | 成本 |
|------|----------|------|------|
| kimi-for-coding | kimi-coding | 主模型，262k上下文，代码优化 | 免费 |
| kimi-k2.5 | bailian | 支持识图，256k上下文 | 免费 |
| qwen3.5-plus | bailian | 备用，262k上下文 | 免费 |
| MiniMax-M2.5 | minimax-cn | 推理能力强，200k上下文 | $0.3/M input |

## 模型切换命令

```bash
# 切换到 Kimi Code（当前主模型）
openclaw config set agents.defaults.model.primary "kimi-coding/kimi-for-coding"
openclaw gateway restart

# 切换到 Kimi K2.5（支持识图）
openclaw config set agents.defaults.model.primary "bailian/kimi-k2.5"
openclaw gateway restart

# 切换到 qwen3.5-plus（备用）
openclaw config set agents.defaults.model.primary "bailian/qwen3.5-plus"
openclaw gateway restart

# 切换到 MiniMax（推理任务）
openclaw config set agents.defaults.model.primary "minimax-cn/MiniMax-M2.5"
openclaw gateway restart
```

## API Key 格式

| Provider | Key 前缀 | 获取位置 |
|----------|---------|---------|
| kimi-coding | `sk-kimi-...` | Kimi 官网 - Coding 模式 |
| bailian | `sk-sp-...` | 阿里云百炼 - Coding Plan |
| minimax-cn | `sk-api-...` 或 `sk-cp-...` | MiniMax 官网 |

## 配置优先级

当模型切换失败时，检查顺序：
1. `~/.openclaw/openclaw.json` — 全局模型配置
2. `~/.openclaw/agents/main/agent/models.json` — 模型定义
3. `~/.openclaw/agents/main/agent/auth-profiles.json` — 认证信息

## 故障排查速查

| 症状 | 可能原因 | 解决方案 |
|------|---------|---------|
| model not allowed | 配置不一致 | 检查两个json文件 |
| API Key 失败 | Key格式错误 | 确认Coding Plan vs 标准接口 |
| 识图失败 | 模型不支持 | 切换到 kimi-k2.5 |
| 网关连接失败 | 端口占用 | `lsof -i :18789` |
