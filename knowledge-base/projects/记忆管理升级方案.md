# 长期记忆管理升级方案

**状态**: 🚧规划中
**标签**: #记忆管理 #系统升级
**更新日期**: 2026-03-04

---

## 当前问题分析

### 1. 上下文压缩不足
- 长对话后 token 消耗大
- 没有自动压缩机制
- 重要信息可能被丢弃

### 2. 记忆检索依赖人工
- 需要手动整理到知识库
- 对话中的关键信息容易遗漏
- 没有自动提取机制

### 3. 跨会话恢复慢
- 模型切换后需要重新读取大量文件
- 没有快速恢复当前会话状态的机制
- 新模型不了解对话上下文

### 4. 缺乏个性化记忆
- 不记录用户偏好和习惯
- 每次都要重新了解工作方式
- 没有情感/风格记忆

---

## 升级方案

### 阶段一：上下文压缩优化

**目标**: 减少 token 消耗，保留关键信息

**实现方式**:
```
对话长度 > 50轮 → 自动触发压缩
保留：
- 用户明确要求的任务
- 重要决策和结论
- 未完成的事项
- 关键数据和时间点

压缩为：
[会话摘要] 3-5句话概括对话内容
[待办事项] 未完成的任务清单
[关键数据] 数字、配置、路径等
```

**文件位置**: `memory/session-summaries/YYYY-MM-DD-HHMM.md`

---

### 阶段二：自动记忆提取

**目标**: 自动识别并保存关键信息

**触发条件**:
- 用户说"记住这个"
- 涉及重要决策
- 记录问题解决方案
- 提及待办事项

**自动分类**:
```
决策类 → decisions/自动提取-YYYY-MM-DD-标题.md
问题类 → problems/自动提取-YYYY-MM-DD-标题.md
待办类 → 更新 memory/YYYY-MM-DD.md 的 TODO 区块
数据类 → 更新 MEMORY.md 的 DATA 区块
```

**实现方式**:
- 对话结束后分析内容
- 使用关键词识别（决策/问题/待办/数据）
- 自动生成摘要和分类

---

### 阶段三：会话状态快照

**目标**: 模型切换后快速恢复上下文

**实现方式**:
```
每10轮对话 → 保存会话快照
文件: memory/session-state/current.json

内容:
{
  "sessionKey": "...",
  "currentTopic": "当前讨论主题",
  "pendingTasks": ["待办1", "待办2"],
  "userMood": "用户当前情绪/状态",
  "lastDecision": "最近的重要决策",
  "keyFiles": ["相关文件路径"],
  "summary": "会话摘要"
}
```

**模型切换后恢复**:
1. 读取 session-state/current.json
2. 3句话告知新模型当前状态
3. 继续对话

---

### 阶段四：个性化记忆

**目标**: 记录用户偏好，提供更个性化服务

**记忆内容**:
```
文件: memory/user-profile.json

{
  "preferences": {
    "documentFormat": "Word",  // 你喜欢 Word 文档
    "modelSwitchStyle": "快捷指令",  // /model k2.5 这种
    "communicationStyle": "简洁直接",
    "responseSpeed": "快速"
  },
  "habits": {
    "activeHours": "21:00-24:00",
    "commonTasks": ["储能测算", "股票分析", "代码开发"],
    "preferredModels": ["kimi-for-coding", "kimi-k2.5"]
  },
  "history": {
    "totalConversations": 100,
    "lastTopics": ["知识库管理", "模型配置"]
  }
}
```

**应用方式**:
- 自动使用偏好的文档格式
- 记住常用指令，提供快捷方式
- 根据活跃时间调整响应风格

## 技术实现

### 需要的脚本
1. `scripts/session-compressor.py` - 会话压缩
2. `scripts/memory-extractor.py` - 自动提取记忆
3. `scripts/state-snapshot.py` - 状态快照
4. `scripts/profile-builder.py` - 个性化档案

### 集成到现有流程
- 每次对话结束 → 触发压缩和提取
- 模型切换前 → 保存状态快照
- 每周回顾 → 更新个性化档案
