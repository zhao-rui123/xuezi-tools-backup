# OpenClaw 灾难恢复手册

**状态**: ✅已生效
**标签**: #OpenClaw #灾难恢复 #紧急
**更新日期**: 2026-03-04

## 适用场景

当 OpenClaw 系统出现以下严重故障时使用：
- Gateway 完全无法启动
- 配置文件损坏或丢失
- 模型切换完全失效
- 飞书连接中断且无法恢复

---

## 故障级别与响应

| 级别 | 现象 | 响应时间 | 操作 |
|------|------|----------|------|
| P0 | Gateway 崩溃，所有消息无响应 | 立即 | 执行完整恢复流程 |
| P1 | 模型切换失败，当前模型可用 | 5分钟内 | 检查配置，重启 Gateway |
| P2 | 单个渠道失效（如飞书） | 30分钟内 | 检查渠道配置 |

## 恢复流程

### 步骤 1: 诊断问题

```bash
# 检查 Gateway 状态
openclaw gateway status

# 查看错误日志
tail -50 ~/.openclaw/logs/gateway.err.log

# 检查配置文件完整性
openclaw doctor --fix
```

### 步骤 2: 备份当前配置（如果可能）

```bash
# 备份当前配置（即使可能损坏）
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak.$(date +%Y%m%d_%H%M%S)
cp ~/.openclaw/agents/main/agent/models.json ~/.openclaw/agents/main/agent/models.json.bak.$(date +%Y%m%d_%H%M%S)
```

### 步骤 3: 从备份恢复

**场景 A: 配置文件损坏**

```bash
# 使用最近的自动备份
BACKUP_DIR="/Volumes/cu/ocu"
cp "$BACKUP_DIR/config/openclaw.json.bak" ~/.openclaw/openclaw.json
cp "$BACKUP_DIR/config/models.json.bak" ~/.openclaw/agents/main/agent/models.json

# 重启 Gateway
openclaw gateway restart
```

**场景 B: 完全丢失配置**

```bash
# 1. 重新初始化配置
openclaw wizard

# 2. 按 MEMORY.md 中的配置重新设置：
# - 主模型: kimi-coding/kimi-for-coding
# - 备选: bailian/qwen3.5-plus, minimax-cn/MiniMax-M2.5
# - 飞书: appId/cli_a928644411785bdf

# 3. 重启 Gateway
openclaw gateway restart
```

**场景 C: Gateway 无法启动**

```bash
# 1. 强制停止 Gateway
openclaw gateway stop
pkill -f openclaw

# 2. 清理临时文件
rm -rf /tmp/openclaw/*

# 3. 检查端口占用
lsof -i :18789

# 4. 重新启动
openclaw gateway start

# 5. 验证状态
openclaw gateway status
```

### 步骤 4: 验证恢复

```bash
# 1. 检查 Gateway 运行状态
openclaw gateway status

# 2. 测试模型切换
echo "测试消息" | openclaw send --model bailian/kimi-k2.5

# 3. 测试飞书连接
# 发送一条测试消息到飞书
```

## 常见问题快速修复

### 问题 1: model 'xxx' is not supported

**原因**: 配置中的模型 ID 和 provider 不匹配

**修复**:
```bash
# 检查当前配置
grep "primary" ~/.openclaw/openclaw.json

# 修正为正确的 provider/model 格式
openclaw config set agents.defaults.model.primary "kimi-coding/kimi-for-coding"
openclaw gateway restart
```

### 问题 2: 飞书收不到消息

**检查**:
```bash
# 1. 检查飞书配置
grep -A3 "feishu" ~/.openclaw/openclaw.json

# 2. 检查 Gateway 日志
grep "feishu" ~/.openclaw/logs/gateway.log | tail -20

# 3. 重启 Gateway
openclaw gateway restart
```

### 问题 3: 模型响应慢/超时

**检查**:
```bash
# 1. 检查网络连通性
curl -I https://coding.dashscope.aliyuncs.com/v1

# 2. 切换到本地模型
openclaw config set agents.defaults.model.primary "minimax-cn/MiniMax-M2.5"
openclaw gateway restart
```
