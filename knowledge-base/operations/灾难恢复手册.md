# OpenClaw 灾难恢复手册

**状态**: ✅已生效
**标签**: #OpenClaw #灾难恢复 #紧急
**更新日期**: 2026-03-06

## 适用场景

当 OpenClaw 系统出现以下严重故障时使用：
- Gateway 完全无法启动
- 配置文件损坏或丢失
- 模型切换完全失效
- 飞书连接中断且无法恢复

---

## 故障级别与响应

| 级别 | 现象 | 响应时间 | 操作 |
|------|------|----------|------|
| P0 | Gateway 崩溃，所有消息无响应 | 立即 | 执行完整恢复流程 |
| P1 | 模型切换失败，当前模型可用 | 5分钟内 | 检查配置，重启 Gateway |
| P2 | 单个渠道失效（如飞书） | 30分钟内 | 检查渠道配置 |

---

- 配置错误 → 我按手册自动修正

## 用户手动恢复指南（P0 故障）

> ⚠️ **使用场景**：Gateway 完全崩溃，飞书消息无响应

### 快速诊断（在终端执行）

```bash
# 1. 检查 Gateway 是否在运行
openclaw gateway status

# 2. 如果显示 not running，尝试启动
openclaw gateway start

# 3. 如果启动失败，查看错误日志
tail -20 ~/.openclaw/logs/gateway.err.log
```

### 常见故障及修复

#### 故障 A: Gateway 进程卡死
```bash
# 强制停止并重启
pkill -f openclaw
sleep 2
openclaw gateway start
openclaw gateway status
```

#### 故障 B: 端口被占用
```bash
# 查看占用 18789 端口的进程
lsof -i :18789

# 杀掉占用进程（替换 <PID> 为实际进程号）
kill -9 <PID>

# 重启 Gateway
openclaw gateway start
```

#### 故障 C: 配置文件损坏
```bash
# 从备份恢复
BACKUP_DIR="/Volumes/cu/ocu"
cp "$BACKUP_DIR/config/openclaw.json" ~/.openclaw/openclaw.json
cp "$BACKUP_DIR/config/models.json" ~/.openclaw/agents/main/agent/models.json

# 重启
openclaw gateway restart
```

#### 故障 D: 完全无法恢复
```bash
# 1. 重新初始化
openclaw wizard

# 2. 按下方"关键配置速查"重新配置
# 3. 重启 Gateway
openclaw gateway restart
```

### 验证恢复

修复后，在飞书发送消息测试：
- 如果收到回复 → 恢复成功
- 如果仍无响应 → 继续排查或等待我上线后处理

## 自动恢复流程（P1/P2 级别）

> 以下流程由我在 Gateway 正常运行时自动执行

### 步骤 1: 诊断问题


## 常见问题快速修复

### 问题 1: model 'xxx' is not supported

**原因**: 配置中的模型 ID 和 provider 不匹配

**修复**:
```bash
# 检查当前配置
grep "primary" ~/.openclaw/openclaw.json

# 修正为正确的 provider/model 格式
openclaw config set agents.defaults.model.primary "kimi-coding/kimi-for-coding"
openclaw gateway restart
```

### 问题 2: 飞书收不到消息

**检查**:
```bash
# 1. 检查飞书配置
grep -A3 "feishu" ~/.openclaw/openclaw.json

# 2. 检查 Gateway 日志
grep "feishu" ~/.openclaw/logs/gateway.log | tail -20

# 3. 重启 Gateway
openclaw gateway restart
```

### 问题 3: 模型响应慢/超时

**检查**:
```bash
# 1. 检查网络连通性
curl -I https://coding.dashscope.aliyuncs.com/v1

# 2. 切换到本地模型
openclaw config set agents.defaults.model.primary "minimax-cn/MiniMax-M2.5"
openclaw gateway restart
```

### 问题 4: Cron 定时任务执行成功但飞书收不到通知

**现象**: 定时任务（如股票推送、健康检查、备份）执行成功，但飞书没有收到通知

**原因**: Cron 环境的 PATH 与 shell 环境不同，导致 `openclaw` 或其他命令找不到

**修复**:

1. **在脚本开头添加完整的环境变量**:
```bash
#!/bin/bash
# 设置环境变量（cron 环境需要）
export PATH="/opt/homebrew/bin:/opt/homebrew/sbin:/usr/local/bin:/usr/local/sbin:/usr/bin:/usr/sbin:/bin:/sbin:$PATH"
export HOME="/Users/zhaoruicn"
```

2. **使用绝对路径或动态检测命令位置**:
```bash
# 方式1: 使用绝对路径
/opt/homebrew/bin/openclaw message send ...

# 方式2: 动态检测（推荐）
OPENCLAW_BIN=$(which openclaw 2>/dev/null) || OPENCLAW_BIN="/opt/homebrew/bin/openclaw"
"$OPENCLAW_BIN" message send ...
```

3. **测试脚本在 cron 环境中是否正常工作**:
```bash
# 模拟 cron 环境测试
env -i HOME=$HOME PATH=/usr/bin:/bin bash /path/to/your/script.sh
```
