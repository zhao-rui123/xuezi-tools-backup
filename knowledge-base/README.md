## 使用规则

### 1. 每日记忆 → 项目知识库
- 原始记忆仍保存在 `memory/YYYY-MM-DD.md`
- 有价值的内容提炼后移入对应项目目录
- 每周回顾一次，整理归档

### 2. 重要决策 → decisions/
- 任何影响后续工作的决定都要记录
- 包括：决策内容、原因、时间、预期效果

### 3. 踩坑记录 → problems/
- 按类别分类（openclaw/云服务器/开发等）
- 记录：问题现象、原因、解决方案、预防措施

### 4. 参考资料 → references/
- API文档、配置指南、数据来源等
- 保持更新，注明最后更新时间
