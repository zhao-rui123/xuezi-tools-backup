# 知识库使用指南

> 所有模型必须阅读并遵循此指南

## 快速开始

### 第一步：读取知识库索引
每次会话启动时，先读取 `knowledge-base/INDEX.md` 了解知识分布。

### 第二步：按需深入
根据当前任务，进入对应目录读取详细文档：
- 做储能项目 → `projects/储能工具包/`
- 排查故障 → `problems/openclaw/`
- 查API配置 → `references/`

## 目录结构规范

```
knowledge-base/
├── INDEX.md              # 全局索引（必读）
├── README.md             # 知识库导航
├── projects/             # 项目知识库
│   ├── 储能工具包/
│   ├── 股票分析系统/
│   ├── 多Agent开发框架/
│   └── 网站运维/
├── decisions/            # 决策记录
├── problems/             # 问题与解决方案
│   ├── openclaw/         # OpenClaw相关问题
│   ├── 云服务器/          # 服务器运维问题
│   └── 开发调试/          # 开发相关问题
├── references/           # 参考资料
├── operations/           # 运维知识（新增）
└── templates/            # 文档模板
```

## 文档格式规范

### 1. 决策记录模板
文件名：`YYYY-MM-DD-决策标题.md`

```markdown
# 决策标题

**日期**: YYYY-MM-DD
**状态**: ✅已采纳 / ⚠️试行 / ❌已废弃
**标签**: #储能 #OpenClaw #运维

## 背景
为什么需要做此决策

## 决策内容
具体决定是什么

## 备选方案
考虑过哪些其他方案

## 预期效果
希望达到什么结果

## 实际结果（后续补充）
决策执行后的实际效果

## 相关链接
- 相关文档
- 相关代码
```

### 2. 问题记录模板
文件名：`问题简述.md`

```markdown
# 问题标题

**状态**: ✅已解决 / ⚠️临时方案 / ❓待验证
**标签**: #飞书 #文件发送 #故障
**发现时间**: YYYY-MM-DD
**解决时间**: YYYY-MM-DD

## 问题现象
具体表现是什么

## 根本原因
为什么会发生

## 解决方案
怎么解决的

## 预防措施
如何避免再次发生

## 验证记录
- [x] 已测试通过
- [ ] 待生产环境验证
```

## 标签系统

### 项目标签
- `#储能` - 储能相关
- `#股票` - 股票分析
- `#OpenClaw` - 系统配置
- `#多Agent` - Agent开发

### 状态标签
- `#紧急` - 需要立即处理
- `#重要` - 高优先级
- `#待办` - 待完成事项
- `#已归档` - 历史记录

### 类型标签
- `#故障` - 问题排查
- `#配置` - 系统配置
- `#流程` - 工作流程
- `#决策` - 重要决定

## 维护规则

### 每日维护
- 检查 `INDEX.md` 是否需要更新
- 确认新增文档已正确分类

### 每周维护
- 回顾 `decisions/`，标记已验证的决策
- 检查 `problems/`，关闭已彻底解决的问题

### 每月维护
- 归档旧文档到 `archive/YYYY-MM/`
- 更新 `INDEX.md` 中的统计信息

## 所有模型必须遵守

1. **读取优先**：回答前先查知识库，不要凭记忆瞎说
2. **及时更新**：发现知识库内容过时，立即更新
3. **规范命名**：新文档按模板格式创建
4. **打标签**：所有文档必须带标签，方便检索
5. **验证状态**：解决方案必须标注验证状态

*此指南所有模型必须遵循*
