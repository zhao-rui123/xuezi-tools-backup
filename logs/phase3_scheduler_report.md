## 🔔 1. 统一通知机制

### 优化前问题
- 多个通知脚本：`feishu_notify.sh`, `feishu_notify.py`, `kilo-notify.sh`
- 通知方式不统一，有些任务没有通知
- 通知级别不明确

### 优化后方案
**统一使用 v4 通知系统**: `/Users/zhaoruicn/.openclaw/workspace/scripts/feishu_notify_v4.sh`

```bash
# 新脚本位置
/Users/zhaoruicn/.openclaw/workspace/scripts/feishu_notify_v4.sh

# 使用方式
./feishu_notify_v4.sh "标题" "消息内容" [级别] [目标]

# 参数说明
- 级别: info|warning|error (默认: info)
- 目标: user|group (默认: group)
```

### 通知机制特点
1. **双重记录**: 同时记录到 notification_log.json 和发送飞书
2. **Kilo广播**: 使用广播专员统一发送
3. **系统日志**: 记录到系统 logger
4. **级别支持**: info/warning/error 三级
