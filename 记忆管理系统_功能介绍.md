# OpenClaw 记忆管理系统 - 功能介绍

> 让 AI 助手拥有长期记忆，越用越懂你

---

## 🎯 解决的问题

普通 AI 对话的痛点：
- ❌ 每次对话都"失忆"，需要重复介绍背景
- ❌ 记不住你的偏好、习惯、项目信息
- ❌ 换了新会话，之前的上下文全丢了
- ❌ 对话太长后，AI 开始"健忘"

这套系统解决以上所有问题。

---

## 📦 四个核心模块

### 1️⃣ Enhanced Memory（增强记忆）
**核心功能：让 AI 记住重要信息**

| 功能 | 说明 |
|------|------|
| **智能存储** | AI 自动判断什么值得记住（偏好/事实/决策）|
| **混合检索** | 向量相似度 + 文本搜索，找得更准 |
| **时间衰减** | 老记忆自动降级，新记忆优先 |
| **重要性加权** | 重要信息永远不忘 |
| **会话提取** | 每次 `/new` 自动总结上次对话 |

**使用场景：**
```
你：我喜欢用深色主题
（AI 存储：用户偏好 - 深色主题）

3天后...
你：帮我做个网页
AI：好的，用深色主题对吗？（自动回忆）
```

---

### 2️⃣ Memory Search（记忆搜索）
**核心功能：随时调取记忆**

| 功能 | 说明 |
|------|------|
| **语义搜索** | 不用关键词，用自然语言找记忆 |
| **多源检索** | 同时搜长期记忆+每日记录+会话历史 |
| **精准定位** | 告诉你在哪个文件、第几行 |
| **关联读取** | 找到片段后，可以读前后文 |

**使用场景：**
```
AI 内心：用户问我之前说的项目...
AI 搜索："用户最近在做什么项目"
AI 找到：3天前 MEMORY.md 第45行
AI 回答：你之前说在做储能收益测算...
```

---

### 3️⃣ Context Management（上下文管理）
**核心功能：让长对话不卡顿**

| 功能 | 说明 |
|------|------|
| **智能压缩** | 对话太长时，自动保留重要信息 |
| **Token 优化** | 减少无效内容，节省 API 费用 |
| **手动压缩** | 随时 `/compact` 清理上下文 |
| **状态监控** | 随时查看当前 Token 使用量 |

**使用场景：**
```
长对话 2小时后...
普通 AI：开始重复、混乱
这套系统：自动压缩历史，保持清晰
```

---

### 4️⃣ Session Persistence（会话持久化）
**核心功能：换模型也不丢上下文**

| 功能 | 说明 |
|------|------|
| **保存状态** | 切模型前自动保存当前工作 |
| **恢复上下文** | 新模型自动了解之前做了什么 |
| **模型切换** | 从 Kimi 换到 Qwen，工作不中断 |

**使用场景：**
```
你：这个模型太慢，换个快的
（切换模型）
新 AI：继续处理你的储能项目...
（自动恢复上下文）
```
