# OpenClaw 知识库管理技能 - 功能介绍

> 告别信息混乱，建立系统化的知识管理体系

---

## 🎯 解决的问题

长期用 AI 助手的痛点：
- ❌ 记忆文件越来越多，找不到了
- ❌ 重要决策散落在各处，忘了为什么这样选
- ❌ 踩过的坑解决了，下次又踩一遍
- ❌ 项目资料东一块西一块，整理困难

这个技能包帮你建立结构化的知识库。

---

## 📚 核心功能

### 1️⃣ 结构化分类存储
**把知识分门别类，不再混乱**

| 分类 | 存放内容 | 示例 |
|------|----------|------|
| **projects/** | 项目知识 | 储能工具包、股票分析系统 |
| **decisions/** | 重要决策 | 技术选型、方案确定 |
| **problems/** | 问题方案 | 故障排查、踩坑记录 |
| **references/** | 参考资料 | API文档、配置指南 |
| **templates/** | 文档模板 | 项目模板、决策模板 |

### 2️⃣ 全局索引系统
**一秒找到想要的信息**

- **INDEX.md** - 所有知识的导航地图
- 按项目/决策/问题/参考分类索引
- 支持标签、时间、关键词检索

### 3️⃣ 自动维护
**每周自动生成维护报告**

| 检查项 | 作用 |
|--------|------|
| 记忆文件统计 | 了解每日记录量 |
| 知识库盘点 | 统计各类文档数量 |
| 归档提醒 | 提醒整理旧记忆 |
| 索引更新检查 | 确保 INDEX.md 最新 |
| 完整性检查 | 确保目录结构完整 |

### 4️⃣ 与 MEMORY.md 协作
**长短记忆结合**

```
短期记忆（MEMORY.md）
    ↓ 每周整理
长期知识库（knowledge-base/）
    ↓ 结构化存储
快速检索（INDEX.md）
```

## 🎁 使用这个技能的好处

| 场景 | 以前 | 现在 |
|------|------|------|
| 找上周的决策 | 翻遍所有记忆文件 | 直接看 decisions/ |
| 项目资料整理 | 散落在各处 | 统一在 projects/项目名/ |

## 📋 工作流程

```
每天：正常对话，记录在 memory/YYYY-MM-DD.md
    ↓
每周：运行维护脚本，生成报告
    ↓
根据报告：整理记忆入 knowledge-base/
    ↓
重要决策 → decisions/
项目进展 → projects/
问题解决 → problems/
参考资料 → references/
    ↓
更新 INDEX.md 索引
    ↓
随时通过索引快速查找
```

## 🔧 安装使用

### 1. 安装
```bash
# 解压到 OpenClaw 技能包目录
tar -xzvf knowledge-base-skill-clean-v1.0.tar.gz -C ~/.openclaw/workspace/skills/
```

### 2. 创建知识库目录结构
```bash
mkdir -p ~/.openclaw/workspace/knowledge-base/{projects,decisions,problems,references,operations,templates}
```

### 3. 初始化索引
创建一个 INDEX.md 作为入口：
```markdown
# 知识库全局索引

## 项目
- [项目名称](projects/项目名/README.md)

## 决策
- [决策标题](decisions/决策文件.md)

## 问题
- [问题描述](problems/类别/问题文件.md)
```

### 4. 配置定时维护
```bash
# 编辑 crontab
crontab -e

# 添加每周维护任务
0 5 * * 1 ~/.openclaw/workspace/skills/knowledge-base/scripts/weekly-maintenance.sh
```

## 💡 最佳实践

### 1. 记录时机
| 类型 | 何时记录 |
|------|----------|
| 决策 | 做出决定后立即记录 |
| 问题 | 解决后立即记录 |
| 项目 | 有里程碑进展时记录 |
| 参考 | 获取资料时立即归档 |

### 2. 命名规范
```
项目：projects/储能工具包/README.md
决策：decisions/2026-03-07-技术选型.md
问题：problems/openclaw/2026-03-07-gateway崩溃.md
```

### 3. 维护频率
- **每日**：正常记录记忆
- **每周**：运行维护脚本，整理入知识库
- **每月**：回顾 INDEX，清理过期内容
