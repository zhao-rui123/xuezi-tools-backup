# 记忆文件系统 + 智能筛选整合方案

## 整合目标

将记忆智能筛选系统（memory-processor）与现有的记忆文件系统深度整合，实现：
1. 每月自动生成记忆摘要
2. 高频关键词同步到知识库
3. 减少未来搜索时的Token消耗

## 工作流程

### 每日流程
```
1. 工作结束 → 保存记忆文件到 memory/YYYY-MM-DD.md
2. 记忆文件永久保留
```

### 每月流程（自动）
```
1. 每月1日 02:00 触发记忆智能筛选
2. 分析上月所有记忆文件
3. 生成月度摘要（memory/summary/YYYY-MM-summary.md）
4. 生成永久记忆存档（memory/permanent/YYYY-MM-permanent.md）
5. 更新关键词索引（memory/index/keywords.json）
6. 更新主题索引（memory/index/themes.json）
7. 同步高频内容到知识库（knowledge-base/INDEX.md）
```

### 查询流程（节省Token）
```
传统方式：
用户提问 → AI搜索所有记忆文件 → 分析内容 → 回答
（消耗大量Token）

优化方式：
用户提问 → AI查知识库索引/月度摘要 → 快速定位 → 回答
（消耗少量Token）
```

## 使用示例

### 场景1：回顾某月工作重点
```
用户：我3月份主要做了什么？

AI：查看 memory/summary/2026-03-summary.md
     快速回答：3月份主要做了技能包开发、系统运维...
     （无需逐篇阅读12个记忆文件）
```

### 场景2：查找特定主题
```
用户：我之前做的零碳园区方案在哪里？

AI：查知识库索引或关键词索引
     快速定位：2026-03月度高频主题包含"零碳园区"
     查看：memory/summary/2026-03-summary.md
           memory/permanent/2026-03-permanent.md
```

### 场景3：长期趋势分析
```
用户：我这几个月都在做什么方向的工作？

AI：查看 memory/index/themes.json
     分析：1月-2月侧重股票分析，3月转向储能项目...
```

## Token节省计算

| 场景 | 传统方式 | 优化方式 | 节省 |
|------|---------|---------|------|
| 回顾某月工作 | 阅读12篇记忆 ≈ 5万Token | 看1篇摘要 ≈ 1千Token | 98% |
| 查找特定主题 | 搜索所有文件 ≈ 3万Token | 查索引 ≈ 500Token | 98% |
| 跨月对比 | 阅读多篇 ≈ 8万Token | 看索引 ≈ 1千Token | 99% |
